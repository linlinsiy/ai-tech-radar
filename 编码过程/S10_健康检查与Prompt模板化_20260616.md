## 详细变更

### 1. main.py — 健康检查替换

**Before**：EIPLite 和 internal_llm 均显示 `"status": "pending"` + `"待集成"` 占位文本

**After**：
- `knowledge_base`：实例化 KBClient → 检查 dataset_id 是否配置 → healthy / degraded
- `internal_llm`：检查 LLM_BASE_URL 是否配置 → 如已配置则调用 call_llm("ping") 验证连通性 → healthy / unhealthy
- 未配置时显示 `degraded`（服务可启动，功能不可用）
- 健康判定逻辑：unhealthy → overall degraded，degraded → overall degraded，全 healthy → healthy

### 2. briefing_job.py — Prompt 模板化

**Before**：system_prompt + user_prompt 均为硬编码 Python f-string 拼接，附带 `# TODO: 从配置文件加载 prompt 模板`

**After**：
- 删除 25 行硬编码 Prompt 字符串
- 替换为 `PromptManager().render_template("briefing", ...)` 
- 模板变量：`briefing_type_cn` / `time_range` / `article_count` / `articles_text` / `insight_count` / `insights_text`
- 模板文件位于 `prompts/templates/briefing.txt`，内容来自 `config/prompts/l5_briefing.properties` 的等效 Jinja2 版本

### 3. prompts/templates/briefing.txt

Jinja2 模板，将系统指令和用户指令合并为单个消息体。结构与 `.properties` 文件中的 prompt 模板一致，使用 `{{ var }}` 语法。
