#### 6.1 EIPLite 客户端正式实现

**产出文件**：[tech-research/internal/app/kb/eiplite.py](../tech-research/internal/app/kb/eiplite.py)

从桩模块升级为正式实现（69 行 → 260 行），包含：

| 功能 | 说明 |
|---|---|
| 认证 | POST /gateway/login → Bearer token，含过期自动续期 |
| 文件上传 | 上传 Markdown 内容 + 标签元数据，返回 file_id |
| RAG 问答 | 检索 + LLM 生成，支持混合/向量/全文三种检索方式 |
| 标签过滤 | 支持 tag_filters 缩小召回范围 |
| 重试策略 | 连接失败/超时/503 自动重试 2 次（10s / 30s 退避） |
| Token 管理 | 自动续期（过期前 60s 刷新） |
| 上下文管理器 | 支持 `with EIPLiteClient(...) as c:` 用法 |

**API 路径约定**：
- 认证：`/gateway/login`（已从 login_resp.json 确认）
- 上传：`/llmpf/api/v1/knowledge-base/files`（默认值，待部署时确认）
- 问答：`/llmpf/api/v1/knowledge-base/chat`（默认值，待部署时确认）
- 全部路径可通过环境变量 `EIPLITE_AUTH_PATH` / `EIPLITE_UPLOAD_PATH` / `EIPLITE_QA_PATH` 覆盖

**工厂函数** `create_eiplite_client()`：从环境变量自动构建客户端实例。

#### 6.2 RAG 问答接口正式实现

**产出文件**：[tech-research/internal/app/api/qa_api.py](../tech-research/internal/app/api/qa_api.py)

从模板占位升级为完整实现（含失败降级逻辑）：

- `POST /api/v1/qa/ask` 完整流程：接收请求 → 创建 EIPLite 客户端 → 认证 → 问答调用 → 返回结果
- 失败降级三层：
  1. EIPLite 不可用 → 503 service_unavailable
  2. 检索返回空 → 200 + "未找到相关内容"提示
  3. 有检索片段但 LLM 生成不可用 → 返回片段原文
- tag_filters 从 QARequest Pydantic 模型自动转换
- 异常安全：EIPLite 连接 `finally` 中关闭
