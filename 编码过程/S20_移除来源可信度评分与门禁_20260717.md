# S20 移除来源可信度评分与门禁

日期：2026-07-17  
范围：AWS L2/L3、内部受控导入与 L4、MySQL 全量 DDL、设计文档和回归测试

## 决策

删除来源可信度评分及其门禁。L3 和 L4 只按统一排序分、L2 的深度分析建议、正文充分性和既有来源/分类均衡规则处理，不再以来源类型推导文章准入资格。

来源名称、类型、域名和原文链接继续保留，用于追溯；它们不再形成评分字段或过滤条件。

## 删除范围

- AWS：删除来源类型到可信度分的映射、L2 输出字段、L3 最低可信度配置和候选门禁。
- 导入：删除受控导入 AnalysisItem 的可信度字段及 AWS 导出字段。
- 内部：删除 ORM 字段、L4 最低可信度配置及选题过滤条件。
- 数据库：从全量建库脚本 ai_radar_article_analysis 删除 score_credibility 列。
- Prompt 和设计：删除要求模型输出可信度的说明，统一评分只保留五个加权维度。

## 数据库影响

新环境直接执行 tech-research/sql/ai_radar_ddl.sql 即可，不会创建该字段。

已有数据库不会自动删除旧列；旧列保留不影响新代码运行。如需物理删除，可在确认不再需要历史字段后执行：

    ALTER TABLE ai_radar_article_analysis DROP COLUMN score_credibility;

## 回归

更新 AWS 和内部 L4 选择器测试，确保高排序分文章不再因来源可信度被过滤。
