# S24 八档领域匹配评分细化

**日期**：2026-07-18  
**范围**：AWS L2 评分、L2 Prompt、设计文档与回归测试  
**不涉及**：HTTP/XXL-Job/crontab 接口、导入协议、数据库表结构、L3/L4 选题逻辑

## 1. 背景

上一版领域匹配度只有 `0/1/3/6/8/10` 六档。标注回归显示，大量普通 AI、Agent、开源或企业文章被模型集中评为 8 分，无法有效区分：

- 直接可用于证券业务或软件研发的内容；
- 有较强可迁移性的工程实践；
- 需要了解但缺乏直接实施细节的模型、框架和生态事件；
- 与本技术岗位关注方向关联较弱的垂直领域内容。

本次保留既有五维评分、权重和 L3/L4 按 `rank_score` 质量优先的机制，仅细化组织相关性评分口径。

## 2. 确认后的评分口径

`score_org_relevance` 固定为 `0/1/3/6/7/8/9/10` 八档，并在 AWS 侧向下归档。

| 分数 | 主要方向 |
|---:|---|
| 10 | 智能投研、交易量化、证券核心系统AI；AI Coding、AI Native SDLC、研发效能平台、Agent工程、Agent/MCP协议工程、企业RAG与知识工程 |
| 9 | 财富管理、证券数据与知识、模型平台与网关、模型服务软件工程、AI数据工程、具有可直接复用软件架构或实施细节的跨行业案例 |
| 8 | 投资银行、风险管理、合规监管、客户与运营；软件测试、AI安全、AI可观测性与治理 |
| 7 | 重大基础模型、模型生态、通用Agent、通用RAG/企业AI/Copilot、重要开发框架/协议/平台、重要开源或开发工具等应知信号 |
| 6 | 国内重大产业/监管、缺少直接工程细节的治理和安全趋势、部分可迁移的非证券金融案例、一般案例、通用基础设施趋势和常规更新 |
| 3 | 迁移价值有限的通用教程、单一云厂商案例或一般跨行业应用 |
| 1 | 无可迁移软件方法的纯GPU/芯片/CUDA、具身智能、机器人、医学、化学、工业视觉等 |
| 0 | 无关、标题党、猎奇或低质量内容 |

补充原则：

- 组织相关性 8 分以上必须由正文明确证明证券业务或具体软件研发机制，不能因出现 AI、Agent、企业、开源等词自动获得高分。
- 非证券金融案例不因“金融”字样自动高分，按实际可迁移性评分。
- 工程参考价值强绑定原厂商、硬件或原行业时通常不高于 4 分；可迁移至证券技术或企业 AI 平台的方法才可达到 7 分以上。
- 趋势重要性 9 分以上必须影响证券 AI、软件研发范式或国内 AI 产业格局；“全球很重要”不等于对本技术雷达重要。

## 3. 代码与配置修改

1. `aws/app/processor/l2_analysis.py`
   - 将领域匹配度归档锚点由六档扩展为八档，保留 7 分和 9 分。
   - 评分公式保持不变：组织相关性 35%、趋势重要性 30%、工程参考价值 20%、技术深度 10%、时效性 5%。
2. `aws/config/prompts/l2_summary.properties`
   - Prompt 升级至 `v3.4`，写入八档方向、工程可迁移性限制和目标读者导向的趋势标准。
   - 明确 `analysis_detail.org_relevance_level`、`org_relevance_reason`、`trend_reason` 为评分审计依据。
   - 评分依据不参与后续 L3/L4/L5 的任何程序逻辑，仅为人工复盘和后续标注校准提供材料。
3. `设计/设计文档_V1.1_20260612.md`
   - 升级至 V1.6，同步八档定义、审计字段用途和工程/趋势评分边界。
4. `aws/tests/test_collection_rules.py`
   - 更新组织相关性归档测试，覆盖 7 分和 9 分锚点。

## 4. 验证

执行 AWS 单元测试、内部单元测试、L2 Prompt 渲染与 Python 编译检查。通过后，重新执行 `scope=rerun` 可使用新 Prompt 重新分析快照；旧分析记录不会自动改分。
