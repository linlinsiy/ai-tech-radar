# S29 评分证据收紧与泛来源语义路由

**日期**：2026-07-19  
**范围**：L1 标题语义路由、L2 组织相关性与工程价值评分约束

## 1. 触发原因

0719 批次共 163 篇 L2 结果、32 篇进入 L3。高分前列已出现 Agent 工程、模型网关和研发实践等目标内容，但仍有部分人物引用、泛软件动态和非目标安全话题获得 8-10 分组织相关性并进入 L3。问题来自 L2 对“AI/Agent/技术博客”等弱信号的过度推断，而不是 L3 容量或来源均衡。

## 2. 改造

1. L2 Prompt 升级为 `v3.5`：8分以上必须由正文同时证明对应领域和具体机制；未给出机制时最高6分；引用、链接聚合、短讯和事实不足内容通常不高于3分组织相关性与工程价值。
2. 对 8 分以上的 `org_relevance_reason` 强制要求写明领域和正文机制/事实，作为人工复盘依据；不新增数据库字段，也不将理由接入后续硬过滤。
3. 标题语义 Prompt 明确：泛软件、人物引用、转发或社会商业话题，不得仅因来源或作者与AI有关而判为相关。
4. 为 Meta Engineering、NVIDIA Developer、Simon Willison、Stratechery 启用 `semantic_routing=true`。AI 垂直官方频道仍可跳过该额外调用，避免扩大无意义模型成本。
5. 标题路由完成后输出候选数、实际路由数、相关数、过滤数、低置信度数和回退数，便于按批次审计。
6. 显式分析 `COL-*` 时忽略历史已处理缓存，只保留批内 URL/正文去重；这样同一月度采集快照可反复用于周度验证和月度正式分析，成功 L2 结果仍写回共享缓存。
7. 显式快照分析的导入统一替代旧 L3 决策；本轮未生成洞察的已分析文章会将旧洞察标记为 `superseded`，避免评分已降低但查询仍显示旧批次“L3已入选”。
8. 明确排除动物福利、消费决策、社会伦理、内容价值观、教育和泛用户体验等 Agent 伦理评测：它们不属于券商软件研发关注的AI安全治理，除非直接涉及企业权限、数据安全、模型治理、开发供应链或金融合规，否则组织相关性最高3分。

## 3. 不变项

- L3 仍使用 `rank_score >= 6.0`，周度容量40、月度/季度容量50；本次不通过提高门槛压缩候选。
- 不恢复来源/分类配额，也不引入绿色通道。
- 分阶段和全流程任务继续共用同一 L1/L2/L3 实现。
