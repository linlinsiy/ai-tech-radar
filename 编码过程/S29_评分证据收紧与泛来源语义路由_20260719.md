# S29 评分证据收紧与泛来源语义路由

**日期**：2026-07-19  
**范围**：L1 标题语义路由、L2 组织相关性与工程价值评分约束

## 1. 触发原因

0719 批次共 163 篇 L2 结果、32 篇进入 L3。高分前列已出现 Agent 工程、模型网关和研发实践等目标内容，但仍有部分人物引用、泛软件动态和非目标安全话题获得 8-10 分组织相关性并进入 L3。问题来自 L2 对“AI/Agent/技术博客”等弱信号的过度推断，而不是 L3 容量或来源均衡。

## 2. 改造

1. L2 Prompt 升级为 `v3.5`：8分以上必须由正文同时证明对应领域和具体机制；未给出机制时最高6分；引用、链接聚合、短讯和事实不足内容通常不高于3分组织相关性与工程价值。
2. 对 8 分以上的 `org_relevance_reason` 强制要求写明领域和正文机制/事实，作为人工复盘依据；不新增数据库字段，也不将理由接入后续硬过滤。
3. 标题语义 Prompt 明确：泛软件、人物引用、转发或社会商业话题，不得仅因来源或作者与AI有关而判为相关。
4. 为 Meta Engineering、NVIDIA Developer、Simon Willison、Stratechery 启用 `semantic_routing=true`。AI 垂直官方频道仍可跳过该额外调用，避免扩大无意义模型成本。
5. 标题路由完成后输出候选数、实际路由数、相关数、过滤数、低置信度数和回退数，便于按批次审计。
6. 显式分析 `COL-*` 时忽略历史已处理缓存，只保留批内 URL/正文去重；这样同一月度采集快照可反复用于周度验证和月度正式分析，成功 L2 结果仍写回共享缓存。
7. 显式快照分析的导入统一替代旧 L3 决策；本轮未生成洞察的已分析文章会将旧洞察标记为 `superseded`，避免评分已降低但查询仍显示旧批次“L3已入选”。
8. 明确排除动物福利、消费决策、社会伦理、内容价值观、教育和泛用户体验等 Agent 伦理评测：它们不属于券商软件研发关注的AI安全治理，除非直接涉及企业权限、数据安全、模型治理、开发供应链或金融合规，否则组织相关性最高3分。
9. Prompt 升级至 `v3.7`：直接证券业务为10分；有具体实现事实的AI Coding、金融数据合成和研发平台为9分；Agent工作流、代码评测、支付金融、低代码和企业AI运营/产品为8分；可迁移安全与伦理内容统一为7分。芯片/RISC-V及融资、AI音乐、CV、消费电子、纯Python、泛AIGC等按1-3分处理；趋势评分改为年度核心10、重要变化7-9、持续方向4-6、小众更新1-3、无价值0，并纳入重大软件工程平台版本影响。
10. L3 选择器增加跨来源趋势确认：在单篇分数门槛前，至少两个不同来源的相似话题会将最高分代表文章的趋势分调整为10并重算综合分。要求代表文章组织相关性不低于6分，因此同源转载及芯片、AI音乐、CV等低相关题材不会因重复报道被提升；提升依据保存到既有 `analysis_detail.topic_trend_boost`，不增加表字段。
11. 修正按时间范围回放快照的筛选：此前发布时间为空的文章会被带入任何周度或月度分析，造成时间范围变化而 L2 数量差异偏小。现在显式范围内只保留发布时间可确认且落在范围内的文章，并将范围外与无发布时间的排除数量写入批次 `date_filter` 统计；无时间范围的最新采集保持原有保留策略。
12. Prompt 升级至 `v3.8`：工程价值改为完整可复用交付物10、完整方法8-9、未强绑定的实践或教程6-7、强绑定实践或公告3-4、无工程内容0-2；趋势改为年度核心硬事件10、已发生重大变化9、重要变化7-8、持续方向/一般更新5-7、观点评论1-3。跨来源趋势提升仅作用于非观点类硬事件，避免人物评论被重复报道后抬升。
13. Prompt 升级至 `v3.9`：组织相关性正式去除“必须提供可直接复用工程细节或证券适配机制”的前置条件，改为按领域匹配度和是否为真实变化事件评分。工程细节只影响工程参考价值和技术深度；高分理由改为说明对应领域和已发生事实。观点类仍固定在3分及以下，非重点硬件、CV、AI音乐等方向的上限不变。
14. Prompt 升级至 `v4.0`：趋势5分被定义为常规、单点、背景参考更新；广泛持续方向为6分，重要安全治理事件为7分，可复用多Agent工程变化为8分，重大模型/API/协议/政策/安全变化为9分，年度或多源确认硬事件为10分。Kimi/Gemma/GPT重大模型发布、Agent数据泄露、提示注入防御和多Agent编排分别补充明确示例，避免不同性质资讯集中落在5分。

## 3. 不变项

- L3 仍使用 `rank_score >= 6.0`，周度容量40、月度/季度容量50；本次不通过提高门槛压缩候选。
- 不恢复来源/分类配额，也不引入绿色通道。
- 分阶段和全流程任务继续共用同一 L1/L2/L3 实现。
