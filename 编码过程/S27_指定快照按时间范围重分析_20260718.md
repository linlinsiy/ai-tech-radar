# S27 指定快照按时间范围重分析

**日期**：2026-07-18  
**范围**：AWS `scope=rerun` 的正式快照回放  
**不涉及**：外部重新采集、历史去重缓存清理、数据库结构、接口验证文档

## 1. 目标

支持从一个已保存的月度正式采集快照中，仅选取近一周文章重新执行 L2 与 L3。该操作用于评分、Prompt 或容量调整后的回归验证，避免再次访问外部来源，也不改变周度增量采集的去重行为。

## 2. 规则

1. `scope=rerun` 配合 `reanalysis_batch_no` 读取指定 `IMP-*` 快照。
2. 可选 `from_date/to_date` 对快照文章按发布时间过滤；未传则回放整份快照。
3. 显式日期范围时，`collection_period=auto` 按该范围解析周度或月度档位，因此近一周回放使用周度 L3 容量 40。
4. 快照中发布时间缺失的文章沿用正式采集时间过滤规则，保留进入回放，避免 RSS/网页缺失日期时无声漏项。
5. 返回值同时记录快照总文章数与实际重分析文章数；范围为空时任务明确失败。

## 3. 验证

- 指定范围没有文章时返回明确错误，不启动 L2/L3。
- 既有指定批次重分析、导入失败持久化及周期档位测试继续通过。
