#### 8.1 QA 接口 Bearer Token 鉴权

设计文档约定内部 API 安全由网络侧防火墙负责，QA 接口新增可选 Bearer token 鉴权层。

**变更文件**：

| 文件 | 变更 | 说明 |
|---|---|---|
| `internal/secrets/.env` | 新增 1 行 | `QA_API_TOKEN=` 占位（不配置则不校验） |
| `internal/app/config.py` | 新增 `qa_config` 属性 | 从环境变量读取 token |
| `internal/app/api/qa_api.py` | 新增 `verify_qa_token` 依赖 | 校验 `Authorization: Bearer <token>`，未配置则跳过 |

**鉴权逻辑**：
- `QA_API_TOKEN` 为空 → 跳过校验（向后兼容）
- 请求头缺 `Authorization` 或非 Bearer 格式 → 401
- Bearer token 不匹配 → 401
- 校验通过 → 放行

#### 8.2 内部侧 XXL-Job Executor

设计文档约定简报生成由 XXL-Job 定时触发（`aiRadarBriefingJob`）。AWS 侧已有 XXL-Job Executor，内部侧补齐对称实现。

**变更文件**：

| 文件 | 变更 | 说明 |
|---|---|---|
| `internal/secrets/.env` | 新增 4 行 | XXL_ADMIN_URL / XXL_APP_NAME / XXL_ACCESS_TOKEN / XXL_EXECUTOR_PORT |
| `internal/app/config.py` | 新增 `xxl_config` 属性 | 从环境变量读取 XXL-Job 配置 |
| `internal/app/jobs/xxl_executor.py` | **新建** | pyxxl 集成，注册 `aiRadarBriefingJob` handler |
| `internal/app/main.py` | 新增 import + 启动代码 | `startup` 事件中以 daemon 线程启动 XXL-Job Executor |

**架构说明**：
- 内部应用节点同时运行 FastAPI 服务（端口 9991）和 XXL-Job Executor（端口 9998）
- XXL-Job Executor 在独立 daemon 线程中运行，与 FastAPI 并行
