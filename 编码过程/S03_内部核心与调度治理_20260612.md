| 5.3 标签过滤 | ⚠️ 桩模块 | 同上，tag_filters 参数已定义，待 EIPLite API 实现 |

**简报生成流程**（`handle_briefing_job`）：
1. 解析调度参数 → 确定时间范围和简报类型（weekly/monthly/topic）
2. 查 MySQL：value_score 降序前 15 篇 + 所有深度洞察
3. 格式化文章和洞察为 LLM 输入文本
4. 调用内部大模型（待配置后启用）
5. 写入 `ai_radar_briefing_draft`（review_status = pending）
6. 生成 Markdown 供 EIPLite 上传
