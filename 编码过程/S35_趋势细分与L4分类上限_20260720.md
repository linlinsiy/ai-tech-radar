# S35 趋势细分与 L4 分类上限

日期：2026-07-20

## 背景

趋势5分聚集了过多单厂商常规更新；L4 虽已限制单来源和资讯类型，但月报仍可能出现一级分类过度集中，例如 Agent 与智能体占 9/14。

## 实现

1. AWS L2 Prompt 升级至 v4.3。趋势5分仅保留两类：有真实上线与可泛化经验的垂直行业 Agent 案例，以及有机制/数据/工程取舍的单点成本、性能或部署优化。
2. 单厂商常规产品/IDE/SDK/运行时更新、单云模型入口、厂商蓝图/教程/运维实践、小版本开源工具、内部工程复盘统一降为趋势3分；观点和营销性单点更新为1-2分。
3. internal L4 新增 `briefing.selection.max_topics_per_category=5`；来源上限3、资讯类型上限5、一级分类上限5均只限制最大数量，不建立最小配额，也不低分补位。
4. L4 完成日志新增 `info_types` 分布，便于直接核验来源、一级分类和资讯类型三类上限。

## 验证

- L4 单元测试覆盖一级分类上限触发、短缺保留和排除原因 `excluded_by_category_cap`。
