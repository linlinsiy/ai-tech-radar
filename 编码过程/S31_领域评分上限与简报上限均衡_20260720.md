# S31 领域评分上限与简报上限均衡

日期：2026-07-20

## 背景

最新月度分析中，视觉 Agent、消费终端等非重点领域因标题包含 Agent、MCP 等词，被 LLM 赋予过高组织相关性；月报的高分主题也存在单一来源集中风险。

## 实现

1. AWS L2 Prompt 升级至 v4.1，明确要求先判断主领域、再判断 Agent/MCP 等技术；计算机视觉、消费终端和硬件主领域不能因包含这些技术词获得高分。
2. AWS L2 在模型评分解析后按文章标题、分类、子分类和技术标签识别明确主领域。计算机视觉、消费终端和硬件主题的组织相关性最高为 3 分；原始分、校正分和原因写入 `analysis_detail.org_relevance_guard`，并打印审计日志。
3. internal L4 保持统一 `rank_score` 排序和同事件聚合，新增两个可配置上限：`briefing.selection.max_topics_per_source=3`、`briefing.selection.max_topics_per_info_type=5`。
4. 上限只会跳过候选，不会建立最小来源/类型配额，也不会用低分主题补足周报、月报或季报目标数。选题审计结果区分 `excluded_by_source_cap`、`excluded_by_info_type_cap` 与 `excluded_by_capacity`。
5. 设计文档已更新为“上限均衡、质量优先”的 L4 规则。

## 验证

- AWS 单元测试覆盖视觉 Agent 的组织相关性上限识别。
- internal 单元测试覆盖来源上限、资讯类型上限，以及达到上限后允许报告短缺而不补低分主题。
