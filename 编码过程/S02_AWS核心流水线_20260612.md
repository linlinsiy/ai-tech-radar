# AI技术趋势雷达 - 编码过程记录

## 第二阶段：AWS 侧核心流水线 L0-L3（2026-06-12 完成）

### 完成状态：✅ 已完成

### 2.1 数据源配置加载
- **状态**：已完成（第一阶段 AWSConfig 已实现）
- **产出**：[aws/app/config.py](../tech-research/aws/app/config.py)
- **说明**：get_data_sources() 方法从 config.properties 读取 12 数据源，解析 `data_sources[N].key` 索引格式，自动过滤 enabled=false 的源

### 2.2 L0 采集层 ✅
| 文件 | 说明 |
|---|---|
| `aws/app/crawler/base.py` | BaseCrawler 抽象基类 + RawArticle 数据类 + CrawlerFactory 工厂 |
| `aws/app/crawler/rss_crawler.py` | **RSS 采集器**：基于 feedparser 解析 RSS/Atom Feed，提取标题、链接、作者、发布时间、摘要，支持白名单域名校验，单源最大采集 100 篇 |
| `aws/app/crawler/web_crawler.py` | **网页采集器骨架**：HTTP GET 获取网页，解析逻辑待按数据源适配（如量子位） |
| `aws/app/crawler/api_crawler.py` | **API 采集器骨架**：REST API 采集，待按具体 API 规范适配 |

**关键设计**：
- RawArticle 使用 Python dataclass，`url_hash` 在 `__post_init__` 中自动计算
- CrawlerFactory 按 `fetch_method` 自动选择采集器类型
- 各采集器独立异常处理，单源失败不影响其他源

### 2.3 L1 解析与去重 ✅
| 文件 | 说明 |
|---|---|
| `aws/app/processor/parser.py` | **ContentParser**：HTML 清洗（BeautifulSoup + lxml）、主体内容提取（article/main 标签优先）、全网获取文章全文（fetch_full_content） |
| `aws/app/processor/dedup.py` | **DedupManager**：基于文件缓存的 URL 去重（processed_hashes.json），支持加载/保存/增量更新 |

### 2.4 LLM 客户端封装 ✅
| 文件 | 说明 |
|---|---|
| `aws/app/llm/client.py` | **LLMClient**：OpenAI 兼容 API 封装，自动重试 2 次（10s/30s），JSON 响应解析（含 ```json``` 代码块提取），调用日志含 token 统计 |
| `aws/app/llm/prompts.py` | **PromptRegistry**：从 config/prompts/ 加载 .properties 文件，render() 方法支持占位符替换，返回 (system, user, version, model) |

### 2.5 L2 基础分析 ✅
- **产出**：[aws/app/processor/l2_analysis.py](../tech-research/aws/app/processor/l2_analysis.py)
- **L2Analyzer**：
  - `analyze_batch()`: 批量并发分析（ThreadPoolExecutor，并发 ≤3）
  - `_analyze_single()`: 渲染 L2 Prompt → 调用 LLM → JSON 解析 → 五维度评分计算
  - `_extract_scores()`: 兼容中英文评分字段名
  - value_score = 五项评分等权平均
  - 单篇失败不阻塞其他文章

### 2.6 L3 深度洞察 ✅
- **产出**：[aws/app/deep/insight.py](../tech-research/aws/app/deep/insight.py)
- **L3Analyzer**：
  - `should_trigger()`: value_score ≥ 8 且有 full_content 时触发
  - `analyze()`: 获取全文 → 渲染 L3 Prompt → 调用高质量模型 → 五维度洞察
  - `analyze_eligible()`: 串行遍历 L2 结果，逐篇判断并执行
  - 自动尝试 fetch_full_content 获取文章全文

### 2.7 受控导入客户端 ✅
- **产出**：[aws/app/exporter/import_client.py](../tech-research/aws/app/exporter/import_client.py)
- **ImportClient**：
  - `build_payload()`: 组装 batch + articles + analyses + insights 四段 JSON
  - `import_batch()`: POST 到内部接口，按设计文档重试策略（10s/30s 退避，最多 2 次）
  - 区分 200/400/413/503/超时/网络异常等错误场景
  - 幂等命中（batch_already_exists）视为成功

### 2.8 编排主流程 ✅
- **产出**：[aws/app/jobs/collect_job.py](../tech-research/aws/app/jobs/collect_job.py)
- **CollectOrchestrator**：
  - `run()`: L0 采集 → L1 去重 → L2 分析 → L3 洞察 → 导入，全链路串联
  - 各阶段统计记录（sources/articles/dedup/l2/l3/import/elapsed）
  - **handle_collect_job()**: XXL-Job JobHandler 入口，解析 scope/sources/from/to 调度参数
  - 支持四种 scope 模式：all / sources / timerange / rerun
