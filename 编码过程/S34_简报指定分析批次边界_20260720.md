# S34 简报指定分析批次边界

日期：2026-07-20

## 问题

原简报生成仅按时间范围读取 L3 成功文章，可能混入使用旧采集范围、旧评分规则或旧正文材料生成的历史洞察。

## 实现

1. `POST /api/v1/jobs/briefing` 新增可选参数 `analysis_batch_no`。
2. 传入时必须匹配 `import_status=success` 的 `IMP-*` 导入批次；不存在或未成功的批次明确失败。
3. L4 查询增加 `ai_radar_article.import_batch_id = 批次ID` 边界，只使用该次分析导入覆盖的成功 L2/L3 文章。
4. 未传参数时保留原有时间范围全量汇总，用于既有定时任务兼容。
5. 选题审计文件和接口返回值记录实际使用的 `analysis_batch_no`。

## 验证

- 导入模型测试覆盖 briefing API 接受 `analysis_batch_no` 参数。
