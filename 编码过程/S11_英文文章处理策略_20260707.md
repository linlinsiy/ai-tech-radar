# S11 英文文章处理策略开发记录

日期：2026-07-07

## 背景

英文技术来源占比较高，若在采集后做全文翻译，会带来较高 Token 成本、较长处理耗时，并可能造成技术术语误译或多译。当前策略调整为：不翻译全文，保留英文原文作为事实源和主召回内容，L2 生成中文标题、中文摘要和标准术语映射，L3 基于原文全文输出中文深度洞察。

## 设计取舍

- 不新增 MySQL 表字段，避免已部署内部库表未迁移时服务启动或导入失败。
- 新增的 `title_cn`、`source_language`、`standard_terms` 随 AWS 到内部侧导入 payload 传递。
- 内部侧在知识库 Markdown 和 `ai_radar_kb_mapping.tags` 中使用这些字段，形成中文入口、英文原文和术语映射的同文档结构。
- 检索层依赖英文原文、中文摘要、标准术语和标签的组合召回，不把 embedding 视为唯一召回方式。
