# S09 知识库与 LLM 模块集成改造

**日期**：2026-06-16
**范围**：internal 应用节点
**关联设计**：设计文档_V1.1_20260612.md

---

## 变更概要

将知识库集成从 EIPLite REST API 模式切换为 TalentsView SDK + 基线工程 LLM 模式。
涉及 4 个文件的修改和 1 个依赖文件的更新。

## 变更清单

| 文件 | 变更类型 | 说明 |
|------|---------|------|
| app/api/import_api.py | 修改 | 集成 KBClient 知识库上传 |
| app/jobs/briefing_job.py | 修改 | 替换 _call_internal_llm 为 call_llm + KB 上传 |
| app/config.py | 修改 | 新增 talentsview_config，扩展 internal_llm_config |
| requirements.txt | 修改 | 添加 talentsview/jinja2/json-repair 依赖 |

## 详细变更

### 1. import_api.py
- 新增 import：create_kb_client / KBError / generate_article_summary / generate_deep_insight
- MySQL commit 后新增 KB 上传步骤（非关键路径）
- 遍历 articles 生成文章摘要 Markdown → upload_file → kb_mapping
- 遍历 insights 生成深度洞察 Markdown → upload_file → kb_mapping
- 返回结果新增 kb_uploaded 字段

### 2. briefing_job.py
- 删除 _call_internal_llm 函数（~100行），改为 call_llm()
- 移除 import httpx / time / InternalConfig
- 新增 KB 上传：简报 Markdown → upload_file → kb_mapping
- 返回结果新增 kb_uploaded 字段

### 3. config.py
- 新增 talentsview_config: app_id / agent_id / workspace_id / kb_dataset_id
- internal_llm_config 扩展: user_id / temperature / max_tokens
- eiplite_config 移除

### 4. requirements.txt
- 新增 talentsview==1.2.2 / jinja2>=3.1.0 / json-repair>=0.25.0

## 技术决策

1. LLM 调用统一使用 llm.client.call_llm()，确保重试策略和 userid header 一致
2. KB 上传为非关键路径，失败不阻断主流程（导入/简报生成）
3. 检索与生成解耦：SDK query() 检索 + call_llm() 生成

## 已知局限

- SDK upload_chunks 当前版本不支持标签参数
- 大批量 KB 上传为同步操作，可能延长 import_api 响应时间
- ai_radar_kb_mapping 的 insight_id / analysis_id 字段当前未写入
